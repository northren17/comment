# Redis GEO 附近商铺功能学习笔记

## 一、Redis GEO 基础

### 1. GEO 是什么

Redis GEO 用于：

存储地理坐标
计算距离
附近搜索

适用于：

* 附近商铺
* 附近的人
* 外卖骑手定位
* 附近停车场

---

## 二、Redis GEO 常用命令

### 1. GEOADD

添加地理坐标：

`GEOADD key 经度 纬度 member`

例如：

`GEOADD shop:geo 116.397128 39.916527 1`

表示：

shopId=1
经度=116.397128
纬度=39.916527

---

### 2. GEODIST

计算两个坐标距离：

`GEODIST key member1 member2 km`

例如：

`GEODIST shop:geo 1 2 km`

返回：

两个店铺距离（km）

---

### 3. GEOSEARCH

附近搜索：

`GEOSEARCH key FROMLOC member BYRADIUS 5 km WITHDIST ASC COUNT 10`

含义：

| 参数       | 作用    |
| -------- | ----- |
| BYRADIUS | 半径搜索  |
| WITHDIST | 返回距离  |
| ASC      | 按距离升序 |
| COUNT    | 返回数量  |

---

## 三、Redis GEO 底层原理

Redis GEO 本质：

GeoHash + ZSet

Redis 会：

经纬度 → GeoHash → score

再利用：

ZSet 排序能力

实现：

附近搜索

---

## 四、附近商铺功能整体流程

用户上传坐标

↓

## 七、GEO 分页核心

### 1. 计算分页参数

`int from = (current - 1) * pageSize;`
`int end = current * pageSize;`

---

### 2. from 含义

跳过多少条

例如：

| 页数 | from |
| -- | ---- |
| 1  | 0    |
| 2  | 5    |
| 3  | 10   |

---

### 3. end 含义

Redis 需要查询前多少条

例如：

| 页数 | end |
| -- | --- |
| 1  | 5   |
| 2  | 10  |
| 3  | 15  |

---

## 八、为什么 Redis GEO 不能真正分页

Redis GEO：

没有 offset

不能：

`LIMIT 5,5`

只能：

LIMIT 前n条

所以：

Redis负责：查询前 end 条
Java负责：skip(from)

实现：

伪分页

---

## 十五、skip(from) 为什么能分页

`all.stream().skip(from)`

作用：

所以：

跳过前N条 = 跳过前几页

## 二十、为什么 Redis 不直接存完整商铺

Redis GEO 更适合：

不适合：

所以：

Redis 查附近

## 二十一、根据 id 查询数据库

---

## 二十二、为什么需要 ORDER BY FIELD

问题：

`WHERE id IN (...)`

不会保证顺序。

Redis 顺序：

5 1 8

表示：

距离顺序

但 MySQL 可能返回：

1 5 8

顺序错乱。

---

## 二十三、ORDER BY FIELD

`last("ORDER BY FIELD(id," + idStr + ")")`

生成：

`ORDER BY FIELD(id,5,1,8)`

作用：

按照 Redis 距离顺序排序

---

## 二十四、FIELD() 是什么

MySQL 函数：

`FIELD(id,5,1,8)`

含义：

id=5 排第一
id=1 排第二
id=8 排第三

---

## 二十五、last() 是什么

`last(...)`

作用：

给 MyBatis-Plus SQL 最后追加原生 SQL

常用于：

* ORDER BY FIELD
* LIMIT
* FOR UPDATE

---

## 二十六、回填 distance

```java

s.setDistance(d.getValue());
```

把 Redis 中的距离重新设置到 Shop 对象

因为：

MySQL 中没有 distance 字段
