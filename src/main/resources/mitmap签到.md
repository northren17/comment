# Redis Bitmap 与黑马点评签到功能复习笔记

---

## 一、什么是 Bitmap

Bitmap 本质：用二进制位（0/1）来表示状态。

```text
1 = 已签到
0 = 未签到
```

### 举例

某用户 5 月前 7 天签到情况：

| 日期 | 1 | 2 | 3 | 4 | 5 | 6 | 7 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| 签到情况 | 是 | 否 | 是 | 是 | 否 | 否 | 是 |

对应 bitmap：

```text
1011001
```

---

## 二、Bitmap 的优点

### 1）非常节省内存

- **普通方式**：每天存一条签到记录，占用空间大
- **Bitmap**：1天只占1bit（1个月仅需约 4 字节）

### 2）查询速度快

因为位运算非常快，适合以下场景：
- 用户签到
- 在线状态
- 点赞状态
- 是否已读
- 打卡记录
- 布隆过滤器

---

## 三、Redis Bitmap 常用命令

### 1）SETBIT

设置某一位的值。

```redis
SETBIT key offset value
```

**示例**：
```redis
SETBIT sign:1:202605 0 1
```

**含义**：用户1在2026年5月1号签到

### 2）GETBIT

获取某一位的值。

```redis
GETBIT key offset
```

**示例**：
```redis
GETBIT sign:1:202605 0
```

**返回值**：
- `1` = 已签到
- `0` = 未签到

### 3）BITCOUNT

统计 bitmap 中 1 的个数。

```redis
BITCOUNT key
```

**作用**：统计累计签到天数

### 4）BITFIELD（重点）

读取一段连续的 bitmap。

```redis
BITFIELD key GET u10 0
```

**含义**：
- 从第0位开始
- 读取10位
- 按无符号整数返回

**应用场景**：连续签到功能核心

## 四、黑马点评签到功能设计

### 1）Redis Key 设计

**格式**：
```text
sign:userId:yyyyMM
```

**示例**：
```text
sign:5:202605
```

**含义**：用户5在2026年5月的签到记录

### 2）Offset 设计

| 日期 | Offset |
| :--- | :--- |
| 1号 | 0 |
| 2号 | 1 |
| 3号 | 2 |
| ... | ... |

**计算公式**：
```java
offset = dayOfMonth - 1
```

## 五、签到功能实现

### 1）用户签到

**Java 代码**：
```java
stringRedisTemplate.opsForValue()
    .setBit(key, dayOfMonth - 1, true);
```

**等价 Redis 命令**：
```redis
SETBIT sign:5:202605 9 1
```

**含义**：5月10号签到（第9位）

## 六、查询功能

### 1）查询今天是否签到

**Redis 命令**：
```redis
GETBIT sign:5:202605 9
```

**返回值**：
- `1` = 今天签到了
- `0` = 今天没签到

### 2）查询累计签到天数

**方法1：使用 BITCOUNT**
```redis
BITCOUNT sign:5:202605
```

**方法2：使用位运算（黑马项目实现）**
```java
int count = 0;
while (num > 0) {
    if ((num & 1) != 0) {
        count++;
    }
    num >>>= 1;
}
```

**作用**：统计二进制中所有 1 的个数，即本月累计签到天数

---

## 七、连续签到（重点）

### 1）核心思想

从今天开始往前数，连续有多少个 1（已签到）。

### 2）为什么要用 BITFIELD

因为需要一次取出本月所有签到位，通过位运算快速统计连续签到天数。

### 3）连续签到代码

```java
int count = 0;
while ((num & 1) == 1) {
    count++;
    num >>>= 1;
}
```

---

## 八、位运算核心理解

### 1）num & 1

**作用**：取二进制最后一位

**示例**：
```text
1011 & 1 = 1
1010 & 1 = 0
```

### 2）num >>>= 1

**作用**：无符号右移一位，删除最后一位

**示例**：
```text
101101
↓
10110
```

---

## 九、累计签到 vs 连续签到（超级重点）

### 1）累计签到

**代码**：
```java
if ((num & 1) != 0) {
    count++;
}
```

**作用**：统计所有 1 的个数

**结果**：本月总签到天数

### 2）连续签到

**代码**：
```java
while ((num & 1) == 1) {
    count++;
    num >>>= 1;
}
```

**作用**：只统计末尾连续的 1

**结果**：连续签到天数

## 十、例子对比（必会）

假设 bitmap 为：`111011`

### 累计签到

统计所有 1：**5 天**

### 连续签到

从最后一位开始：
```text
1 √
1 √
0 停止
```

结果：**2 天**

## 十一、BITFIELD 返回值为什么是 List

因为 BITFIELD 支持多个 GET 操作。

**示例**：
```redis
BITFIELD key
GET u8 0   # 获取前8位
GET u8 8   # 从第8位开始获取8位
```

**返回值**：`[12, 25]`（每个 GET 对应一个结果）

所以 Spring Data Redis 统一返回 `List<Long>`。

---

## 十二、为什么使用 result.get(0)

在签到场景中，我们只执行了一个 GET 操作：

```java
List<Long> result = stringRedisTemplate.opsForValue().bitField(
    key,
    BitFieldSubCommands.create()
        .get(BitFieldType.unsigned(dayOfMonth)).valueAt(0)
);
// result = [718]

Long num = result.get(0);  // num = 718
```

## 十三、布隆过滤器与 Bitmap（扩展）

### 核心思想

Bitmap + 哈希算法，用于判断元素是否存在于集合中。

### 哈希映射

```java
int index = id % bitmap.size;
```

**作用**：把超大 ID 映射到 bitmap 的范围之内。

### 示例

假设 bitmap 长度为 10：

```java
int userId = 23;
int index = 23 % 10;  // index = 3
```

### 哈希冲突

```java
int userId1 = 23;  // 23 % 10 = 3
int userId2 = 33;  // 33 % 10 = 3
```

两个不同用户映射到同一位，可能导致误判。

### 布隆过滤器结论（必背）

- **能保证**：不存在的数据一定不存在
- **不能保证**：存在的数据一定存在（可能发生哈希冲突）

## 十五、常见面试题

### 1）Bitmap 相比普通存储方式有什么优势？

**答案**：
- **空间效率**：Bitmap 用位存储，1个月签到仅需约 4 字节，相比传统的字符串存储节省大量空间
- **查询效率**：位运算非常快，统计签到天数只需简单的位操作

### 2）如何实现连续签到功能？

**答案**：
1. 使用 BITFIELD 获取本月所有签到位
2. 通过位运算从最后一位开始统计连续的 1
3. 遇到 0 时停止，统计结果即为连续签到天数

### 3）布隆过滤器的特点是什么？

**答案**：
- **优点**：空间效率高，查询速度快
- **缺点**：存在误判率（不存在的数据一定不存在，存在的数据可能误判为存在）
